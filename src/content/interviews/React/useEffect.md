# React useEffect 面试逐字稿

## 面试官
useEffect 的依赖数组分别代表什么？为什么很多人会写出问题？

## 回答思路
可以分三种情况回答：

1. 不传依赖数组：每次渲染后都会执行。
2. 传空数组：只在初次挂载后执行一次。
3. 传具体依赖：依赖变化后重新执行。

## 进一步说明
真正容易出错的点不是 API 本身，而是副作用和闭包。比如 effect 里读到的是旧状态，或者忘了清理订阅、定时器、事件监听。

## 面试时可以补一句
如果 effect 只是为了根据 props/state 计算数据，通常不应该写 effect，而应该直接在渲染阶段推导。

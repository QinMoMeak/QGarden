# JavaScript 闭包高频问答

## 面试官
请你解释一下什么是闭包，以及闭包在项目里的真实用途。

## 回答思路
闭包本质上是函数和它声明时所处词法作用域的组合。即使外层函数已经执行结束，内部函数依然可以访问外层作用域中的变量。

## 可以展开的重点
1. 先解释词法作用域。
2. 再说明“函数执行结束后变量为什么还在”。
3. 最后落到实际用途，例如缓存、封装私有状态、函数工厂。

## 示例
```ts
function createCounter() {
  let count = 0;

  return () => {
    count += 1;
    return count;
  };
}
```
